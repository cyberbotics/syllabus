  • Task (Steer based on obstacle distance)Open or Close
    1. The first thing to do is to retrieve the values from those additional four sensors within each loop iteration.
      1. You will find the sensor names starting on line 21 of the code.
    2. Rather than alter the wheel speed by a fixed value, this time, we will do this dynamically by using the distance sensor values.
      1. The closer you are to an obstacle, the sharper you want your turn to be.
    3. There are now multiple sensor values, so we need to combine those somehow. Let us use an average.
    4. Delete the if statement and its body and replace it with the following two lines of code. 
    
lhs_motor.setVelocity(3 - (mid_rhs_gap + rhs_gap) / 2)
rhs_motor.setVelocity(3 - (mid_lhs_gap + lhs_gap) / 2 - mid_gap)
    1. The left-hand motor velocity is reduced based on an average of the right-hand front-facing distance sensors.
      1. If an obstacle is detected to the front right, the left motor reduces speed, and the robot turns left.
      2. The same logic applies to the right-hand motor, but this line of code also includes the centre front sensor value.
    2. Why is that?
      1. If an obstacle is detected straight ahead, it does not matter whether you veer to the left or right. In this case, we have chosen to turn right.
    3. Save the code (CTRL + s, or File->Save).
      1. Reset the simulation by pressing the reset button in the toolbar of the simulation window and then pressing play.

    Expected Result

    The robot will now turn when an obstacle is ahead. However, you may observe that the motion is jerky and some collisions still occur.

  • ExplanationOpen or Close

    The values from the distance sensors have a completely different range than the velocity values. Therefore you may exceed the maximum allowed velocity value. As the motor cannot achieve this velocity it defaults to the maximum 9.53 radians per second top speed. Therefore the robot's motion will be jerky.

  • Task (Calibration)Open or Close

    Adjusting the values to keep them within an allowed range is called calibration. Let us calibrate the velocity values by dividing the distance sensor values by a calibration constant before setting the velocity. The constant DIST_SENSOR_CAL_CONST has been created for you and is set as 360.

    1. Edit your code as follows: 
    
mid_gap= mid_sensor.getValue()/DIST_SENSOR_CAL_CONST
mid_rhs_gap = mid_rhs_sensor.getValue()/DIST_SENSOR_CAL_CONST
rhs_gap = rhs_sensor.getValue()/DIST_SENSOR_CAL_CONST
lhs_gap = lhs_sensor.getValue()/DIST_SENSOR_CAL_CONST
mid_lhs_gap = mid_lhs_sensor.getValue()/DIST_SENSOR_CAL_CONST
    1. Save the code (CTRL + s, or File->Save).
    2. Reset the simulation by pressing the reset button in the toolbar of the simulation window and then pressing play.
